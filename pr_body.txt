DOC-MECH-FRESHNESS-GENERATOR-API-STRICT-S2

This PR makes the Claim Evidence Map generator validate the Lenskit freshness bridge not only through the CLI path, but also when generate(repo_root=...) or check(repo_root=...) are called directly as Python APIs.

Why:
main() already validated before CLI generation/check. Direct API calls could still load an invalid registry and render/compare output. This closes that bypass and preserves the invariant: no claim evidence map generation/check without a valid freshness registry.

Scope:
- generate() validates before reading/rendering/writing
- check() validates before reading/comparing
- tests cover invalid bridge registry rejection through both public helpers
- no JSON artifact is reintroduced
- no wall-clock freshness computation is reintroduced
- no claim scope or registry semantics changed
